Q: Explain the difference between A/B testing and Multivariate testing in Adobe Target, using Panera's mobile app as an example.

Simple Explanation:

Think of A/B testing like comparing two completely different restaurant menus. You show half your customers Menu A and half your customers Menu B, then see which one sells more food.

Multivariate testing is like testing different parts of the same menu at once - different colors, different fonts, different pictures, and different prices - all mixed together to see which combination works best.

🍞 A/B Testing Example - Panera App Homepage
  • Version A: Shows "Order Now" button prominently at top
  • Version B: Shows menu categories first, then order button
  • Result: See which gets more people to start ordering

When to use: Testing big changes like complete page redesigns

  • 2 Images: Salad photo vs. Sandwich photo
  • 2 Buttons: "Add to Cart" vs. "Order Now"

When to use: Testing small changes on elements that work together

Q: How do you determine statistical significance and sample size for A/B tests?

Sample Answer:

Statistical significance and sample size are critical for reliable test results. Here's my approach:

Pre-test Planning:

  • • Set significance level (typically 95%, α = 0.05)
  • • Define statistical power (typically 80%)
  • • Determine minimum detectable effect (MDE)
  • • Calculate required sample size using these parameters

Sample Size Formula Factors:

n = 2 × (Zα/2 + Zβ)² × p × (1-p) / (δ)²
Where: p = baseline conversion rate, δ = effect size

During Test:

  • • Monitor for adequate sample size before drawing conclusions
  • • Use confidence intervals to understand result ranges
  • • Avoid peeking issues by pre-defining analysis schedule

Best Practice: Never stop a test early based on statistical significance alone - always reach your pre-calculated sample size.
